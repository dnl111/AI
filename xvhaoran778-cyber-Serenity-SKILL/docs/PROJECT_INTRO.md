# 项目简介

`serenity-chokepoint-investing` 是一个面向 Codex 的投资研究 skill。它把 Serenity 公开材料中反复出现的研究动作，整理成一个跨市场可复用的工作流：

1. 从终端需求冲击出发，而不是从股票代码出发。
2. 沿产业链向上游拆解 BOM、材料、设备、封装、产能、资格认证和政策约束。
3. 找出真实卡点：供给集中、扩产慢、替代难、架构绑定、政府或客户验证强。
4. 往下至少挖三层：从 GPU/HBM/电力等显性瓶颈，继续挖到激光器、衬底、晶体生长、测试、封装、光纤、电缆等二阶/三阶卡点。
5. 判断同一个需求冲击对不同体量公司的财务影响是否存在巨大不对称。
6. 识别小市值定价真空：低覆盖、低流动性、机构无法买够、空头挤压只作为加速器。
7. 用贝叶斯方法持续更新：先验、证据、反证据、后验方向。
8. 区分“交易披露”和“观点表达”，避免把看多内容误当成交记录。

## 为什么要加入公司新闻和公告抓取

需要加入，而且是这个 skill 的核心能力之一。

Serenity 风格的研究不是静态估值模型，而是动态证据更新系统。公司公告、财报、客户订单、补贴、扩产、融资、减持、监管、短报告和媒体报道，都会改变“这家公司是否真是瓶颈”的后验概率。

因此 skill 中新增了 `market-data-and-news.md`，明确规定：

- A 股优先看交易所公告、巨潮资讯、互动易/上证 e 互动、政策文件。
- 美股优先看 SEC EDGAR、公司 IR、earnings call、8-K、S-3/ATM、short reports。
- 港股优先看 HKEXnews/披露易、年报/中报、配售、关连交易、内幕消息公告。
- 台股、日股、欧洲股票优先看当地交易所披露、公司 IR、政府补贴和客户验证资料。

新闻不是结论，只是证据流。Skill 会把新闻和公告分成：

- 证实假设
- 削弱假设
- 中性噪音

## 适用场景

- 分析一个具体股票是否有供应链瓶颈机会。
- 比较同一产业链上多个候选标的。
- 把某个技术趋势转译成公司收入、利润、订单或估值影响。
- 对已有投资假设做贝叶斯更新。
- 整理某个投资博主或研究员的公开交易披露。

## 不适用场景

- 直接跟单。
- 只根据社交媒体热度选股。
- 不看公告和财报就下结论。
- 把 Serenity 的公开收益率当作可验证审计结果。
